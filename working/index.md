# Working Notes

This folder is the persistent running log for the broader builder work in this PR.

Use it to record:

- work that has been completed
- important code and contract changes
- reasoning behind decisions
- blockers, deferrals, and tradeoffs
- context the next plan or implementation slice needs
- key rules and architectural guardrails that future slices must preserve

This folder is intentionally persistent.

- Keep updating `working/notes.md` as the builder work progresses.
- Keep `working/handover.md` up to date with important rules, layering decisions, scope boundaries, and other guidance that must survive beyond transient plan folders.
- Do not delete this folder during normal plan cleanup.
- Keep it until the user explicitly says the broader builder effort is complete.
